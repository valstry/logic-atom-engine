   - 对照优化

### 2.1 大目标
跑通「逻辑原子体系」本身，证明它是可编排、可复用、可扩展的方法框架。

### 2.2 当前小目标（展示场景）
在大目标下，先把「消除知识壁垒」这个场景跑明白：
- 把专业文本中的隐性前置知识挖出来
- 逐层补齐并降阶表达
- 让普通人也能读懂

### 2.3 插件角色定位
- 插件是“梁和柱”（通用框架）
- 场景是“装修风格”（可替换应用）

## 4. 当前核心问题（必须继续解决）

1. 仍有“看起来可用、实际不自洽”的部分  
   - 工作区“显示层”和执行策略“方法层”还存在耦合混乱。

2. 「消除知识壁垒」方法链还不完整  
   - 还没有严格落成“原文/笔记对照 -> 压缩迭代 -> 受约束提问/一步推断”的闭环。

3. 流程调序虽然有入口，但需进一步验证端到端语义  
   - 调序后必须影响真实执行顺序，而不是只改 UI。

4. 401 问题需要彻底解决，而不是依赖降级来掩盖  
   - 目标是全功能正常执行。

## 5. 方法链标准（本项目必须对齐）

「消除知识壁垒」在插件里应对应以下可执行链：

1. 结构化阅读：抓对象、定义、描述、关系、动作  
2. 隐性知识抽取：识别原文默认前提  
3. 知识差分：已知 / 新知 / 冲突 / 待验证  
4. 补丁注入：把前置知识补到原文理解路径  
5. 压缩笔记：形成最小记忆单位  
6. 对照优化：笔记与原文校验并继续压缩  
7. 高阶应用：基于短笔记做受约束提问与一步推断（非发散）

---

## 6. 逻辑原子与方法步骤映射原则

关键原则：  
不是“逻辑原子去模拟读写算”，而是“用平台基础动作执行逻辑原子，再由逻辑原子拟合方法步骤”。

可用映射方向（当前版本）：
- `decompose`：抽取对象/定义/关键词/隐含前提
- `associate`：关系链、差分判断、一步推断
- `transform`：降阶改写、补丁注入
- `abstract`：结构化压缩、记忆单位生成
- `evaluate`：优先级、下一步动作选择
- `iterate`：可读性/准确性迭代

## 8. 验收清单（DoD）

- 调整流程顺序后，一键执行的阶段顺序同步变化
- 每个阶段输出被拆成可操作点卡片
- 点卡片可复制/插入，并可写入画像记忆
- “对照优化”阶段可见：指出过长/过简/保留项 + 给出更短版本
- 高阶应用输出受约束：只做一步推断，问题必须与短笔记定义直接相关
- 出现连接问题时能给出可修复路径；配置正确时全流程可正常跑完
